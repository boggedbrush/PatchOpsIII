Quick start & essentials

PatchOpsIII is designed to be simple: point it at your BO3 install, apply the patches you need, and keep a clean log for troubleshooting.

Get Help

Quickstart

Windows

Windows Download
  1. Recommended: Move PatchOpsIII to a folder in your Desktop or Documents.
  2. Launch PatchOpsIII.
    • If Black Ops III is installed in the default Steam location, it will be detected automatically.
    • Otherwise click Browse and select your Black Ops III folder (it will be saved).
    • Alternative: move PatchOpsIII into the same directory as your game to auto-detect it.
Show default Steam install path (Windows)

C:\Program Files (x86)\Steam\steamapps\common\Call of Duty Black Ops III

Success check: PatchOpsIII opens and shows your Black Ops III install.

Linux / Steam Deck

AppImage Download
  1. Recommended: Install it with Gear Lever for automatic updates.
    • Right-click the AppImage → Open With → Gear Lever
    • Click Unlock
    • Click Move to app menu
  2. Launch PatchOpsIII.
    • If BO3 isn’t auto-detected, click Browse and select your game folder (same as Windows).
Manual install (no auto-updates)

If you don’t want to use Gear Lever, you can run the AppImage manually — check the download page for manual instructions.

Show default Steam install path (Linux)

/home/$USER/.steam/steam/steamapps/common/Call of Duty Black Ops III/

Success check: PatchOpsIII opens and shows your Black Ops III install.

Troubleshooting

Check logs first

PatchOpsIII generates a PatchOpsIII.log file and prints live status messages in the embedded terminal.

Launch options are sensitive

Launch option behavior can vary across systems. Apply changes incrementally and confirm each option is stable before stacking more.

Config safety

If you use the read-only toggle, remember to disable it before making manual edits to config.ini.

FAQ

Do I need to keep t7patch running in the background?

PatchOpsIII’s T7 Patch management is designed to apply the patch and configuration cleanly; it does not require you to keep a separate patch process open for basic setup workflows.

Is DXVK required?

No. DXVK-GPLAsync is optional, but it can reduce shader compilation stutter by enabling async pipeline compilation through Vulkan.

Where can I request features or report issues?

Use GitHub Issues: github.com/boggedbrush/PatchOpsIII/issues.
